Task Manager Project Summary

Date: October 24, 2024

Overview

This project is a Task Manager application built with Node.js, Express, and MongoDB. It allows users to create, read, update, and delete tasks.

Major Learnings

  • Arranging code in different folders
  • Creating and handling routes
  • Using middlewares
  • Defining and using models
  • Connecting to a database
  • Error handling
  • Writing controllers
  • Setting up environment variables using .env
  • Linting code with ESLint
  • Using Postman (creating collections, global variables, sending requests)

Functionality

  • Get All Tasks: Fetch all tasks from the database using the GET /tasks endpoint. Implemented in the getTasks function.
  • Create Task: Create a new task using the POST /tasks endpoint. Implemented in the postTasks function.
  • Get Task by ID: Fetch a single task by its ID using the GET /tasks/:id endpoint. Implemented in the getId function.
  • Update Task: Update a task by its ID using the PATCH /tasks/:id endpoint. Implemented in the patchId function.
  • Delete Task: Delete a task by its ID using the DELETE /tasks/:id endpoint. Implemented in the deleteId function.

Middleware

  • Async Wrapper: A middleware to handle asynchronous functions and avoid repetitive try-catch blocks. Defined in asyncWrapper.
  • Error Handler: A middleware to handle errors and send appropriate responses. Defined in errorHandler.

Database Connection

The project connects to a MongoDB database using Mongoose. The connection logic is defined in connectDB.

Models

The Task model is defined using Mongoose schema in Task.js.

Routes

The routes for the task-related endpoints are defined in tasks.js.

Environment Configuration

Environment variables are managed using a .env file, which is ignored by Git as specified in the .gitignore file.
